## 議論したこと（トピック）

- 同種サービス（Supabase / Firebase / Appwrite / PocketBase / さくらインターネットのみ構成 等）の比較。
- フロントを **Next.js（Vercel）** にするか **React + Vite** にするか → **SEO 不要・Turnstile 検証を同リポジトリで扱いやすい**点から **Next.js** をやや優先する結論。
- **X（Twitter）の OAuth／書き込み API**は規約・料金の変動が大きく、**ログイン連携も自動ポストも採用しない**方針に寄せた。
- **問い合わせ**：メール露出より、**告知用 X アカウントへの @メンション**を主手段にする案で整理。
- **新規投稿の運用**：**Discord Incoming Webhook** に「X にコピペしやすい文案」を飛ばし、**手動**で告知アカウントからポストする（無料寄り）。

## 決めたこと（確定事項の要約）

### インフラ・技術

| 区分 | 内容 |
|------|------|
| フロント | **Next.js**（App Router 想定）を **Vercel**（Hobby 無料枠前提）へデプロイ。 |
| バックエンド | **Supabase**（PostgreSQL、PostgREST、RLS、Storage、**管理者向け Auth**）。 |
| bot 対策 | **Cloudflare Turnstile** を **Route Handler** 等で検証してから書き込み。 |
| SEO | **優先しない**。 |

### プロダクト・仕様（機能仕様書に反映済み）

- **背景**：旧 nicoevents 統合後の使い勝手、後援会の負荷感 → **非公式の掲示板的**カレンダー。
- **スコープ**：当面 **車載動画**系の投稿祭のみ（将来のカテゴリ拡張はコード上の伸びしろ程度）。
- **投稿者認証**：**案 A** — **ログインなし**。**投稿者表示名**と **X 識別子**は**いずれも必須・手入力**。
- **必須項目**：タイトル、本文（プレーン＋改行）、開催開始日・終了日（**日付のみ**、基準は **JST**）、告知 URL（http/https）、サムネ 1 枚（**最大 5MB**、JPEG/PNG/WebP）、表示名、X 識別子。
- **バリデーション**：**開始日 ＞ 終了日**はエラー。開始が今日より前は**許可＋警告**。極端に長い期間は**許可＋警告**（しきい値は実装時）。
- **一覧**：トップ＝**未終了のみ**、**開催開始日の降順**。**終了分は別ページ**。**開催前／中／後**の表示。初版は**ソート切替なし**。
- **編集・削除**：初版は**編集なし**。**管理者のみ全件削除**。本人削除は**なし**。
- **投稿フロー**：**入力 → 確認画面（要約＋編集不可の注意＋問い合わせ案内）→ 確定**。
- **Discord**：投稿**保存成功後**に Webhook 通知。**失敗しても利用者の投稿はロールバックしない**。
- **問い合わせ**：**手動投稿用／告知用 X アカウント**への **@メンション**を**主**。サイト内フォームは初版**必須としない**（補完として後から可）。
- **UI**：シンプル・長時間見ても疲れにくい。**コスト・手間**は可能な限り抑える（セキュリティ必須分は除く）。
- **静的ページ**：利用規約、使い方、問い合わせ導線。
